重复引用错误-Duplicate files copied in APK

原因：

在使用umeng推送时出现了这个问题： 

Duplicate files copied in APK META-INF/maven/com.squareup.okio/okio/pom.xml

原因是okhttp和 umeng推送 都使用到okio

解决方法：

你可以在主项目的build.gradle中添加下面的内容，排除重复的东西：

android {
    ....

    packagingOptions {
        exclude 'META-INF/maven/com.squareup.okio/okio/pom.xml'
        exclude 'META-INF/maven/com.squareup.okio/okio/pom.properties'
    }
}

在google的gradle文档中写着：

Configuration blocks（配置块）
Listed below are the configuration blocks available within android（下面列出的是在Android的可配置模块）

个人理解为build.gradle中android的内容配置模块：

android {
....
}
